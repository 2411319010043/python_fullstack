# 中间件

## 函数中间件

### what

’中间件‘是一个函数，在每个**请求**被特定的路径操作处理之前，以及每个**响应**之后工作

### how

1. 定义一个函数 加上 @app.middleware(“http”)装饰器
2. 
   * **参数 `request: Request`**：当前的 HTTP 请求对象。
   * **参数 `call_next`**：这是一个函数，调用它 (`await call_next(request)`) 会将请求传递给“下一个”中间件或最终的路由。
   * **返回值**：必须返回一个 `Response` 对象（通常是 `call_next` 返回的响应，经过修改后）。

#### 示例

```python
import time
import uvicorn
from fastapi import FastAPI, Request

app = FastAPI()

@app.middleware("http")  # 添加装饰器，将下一行的函数注册为中间件
async def add_process_tiem_header(request: Request, call_next):
    # 请求处理前：记录开始时间
    start_time = time.time()
    response = await call_next(request)  # 将请求传递给“下一层”(可能是其他中间件或最终的路由)
    # 在生成响应数据返回之前执行
    process_time = time.time() - start_time  # 计算耗时
    response.headers["X-Process-Time"] = str(process_time)  # 修改响应头
    return response  # 返回响应

if __name__ == '__main__'：
	uvicorn.run(app=app)
```

### why

中间件只需写一次，自动应用于所有路由。

## 类中间件

### what

类中间件是将中间件逻辑封装在类中

### how

1. 基础模式：实现__**`__call__`**__方法
2. 纯ASGI类中间件模式
   * 导入
   * 实例化
   * 初始化
   * 一个根路由
   * 主函数(启动应用)
3. 带依赖注入的类中间件

#### 完整的必备流程

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
import uvicorn

app = FastAPI()

app.add_middleware(
	CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
  	allow_headers=["*"]
)

@app.get("/")
def home():
    return {"status": "healthy"}

if __name__ == '__main__':
    uvicorn.run(app=app)
```



### why

相比于函数中间件，类中间件功能完整，配置即可用
