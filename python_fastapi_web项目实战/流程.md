# 流程

#### 前端(发请求，接收响应) → router接口（API）(接收前端请求，对请求的参数进行校验，调用services)→ services(接收接口校验好的参数 增删改查) → models.py(用orm写出来的数据库表，映射规则) → 数据库

#### models 定义的是表结构,表在数据库里已经存在了,那 models 还会不会再执行？

	会 services不能绕过models直接操作数据库 
	models 定义了 Python 对象与数据库表之间的映射关系， ORM 根据这个映射关系，将 services 中的操作翻译成 SQL 并执行。
	
	models：规则
	services：意图
	ORM：翻译 + 执行

##### models.py只是一张说明书 没有任何业务操作 什么都不执行
