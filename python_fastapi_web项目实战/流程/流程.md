# 流程

#### 前端(发请求，接收响应) → schemas(请求模型校验) → router接口（API）(接收前端请求，对请求的参数进行校验，调用services)→ services(接收接口校验好的参数 增删改查) → models.py(用orm写出来的数据库表，映射规则) → 数据库

**schemas = 数据长什么样**
 **router = 入口 / 出口**
 **services = 干什么事**
 **models = 存成什么样**

#### models 定义的是表结构,表在数据库里已经存在了,那 models 还会不会再执行？

	会 services不能绕过models直接操作数据库 
	models 定义了 Python 对象与数据库表之间的映射关系， ORM 根据这个映射关系，将 services 中的操作翻译成 SQL 并执行。
	
	models：规则
	services：意图
	ORM：翻译 + 执行

##### models.py只是一张说明书 没有任何业务操作 什么都不执行

## 建模顺序

​	schemas → models → services → router → init
​	数据长什么样 → 存成什么样 → 怎么用 → 给谁用

#### 示例登记模块

1. shcemas.py
   * 前端会传什么？
   * 前端能看到什么？
     - 接口输入格式
     - 接口输出格式
     - 字段是否必填
2. models.py
   * 这些数据要怎么存进数据库？
     - 可以多字段
     - 可以加索引/唯一约束
     - 不影响接口
3. services.py
   * 我需要对登记模块做哪些事？
     - 新增
     - 查询
     - 列表
     - 删除
4. router.py
   * 前端通过什么URL，什么方法来用这些功能？
5. __init__.py
