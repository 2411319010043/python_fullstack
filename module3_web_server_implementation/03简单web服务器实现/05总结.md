tcp是基于三次握手四次挥手通信的
握手：1.客户端发送请求 请服务器准备好资源
     2.服务器回复请求 + 服务器请客户端准备好资源
     3.客户端回复请求
     ...传输开始

挥手：1.客户端调用.close() 告诉服务器 不再发送数据
     2.服务器接收到数据后 立马回复客户端 确认接收到客户端不在发送数据
     3.服务器看还有没有数据要发给客户端 如果没有数据了 就调用.close() 并告诉客户端 不再发送数据
     4.客户端确认接收到数据 然后等待一段时间 以防服务端没有接收到数据
     (5).如果服务器并没有接收到客户端发送的确认信息 那么一段时间后会重新发送
