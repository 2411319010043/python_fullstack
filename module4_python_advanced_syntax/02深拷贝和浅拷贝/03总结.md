一、网络编程
    1. 两台电脑可以通过网线直接通信 三台及以上的只能通过集线器或交换机通信 这是因为网线传输的是数据信号 不是电信号
    第一代 集线器 通过广播的方式通信 效率低 主机多了会堵塞
    第二代 交换机 可以单播 广播
    网络掩码是为了区分出 网络号和主机号

    2. 第一次通信是如何获取对方mac地址的？
        每个主机都有独立的mac地址 通过arp获取mac地址 浏览器发送全F地址(一个通用地址 所有的主机除了接收自己mac地址外 还会接收这个地址) 交换机广播后 服务器收到后 去掉mac地址 通过查看 目的ip地址是不是自己的 来区分 是否是要本服务器的mac地址

    3. 交换机只能连接有限个主机 组成小网 是如何实现跨网通信的？
        通过路由器  多个主机 ---> 交换机 ---> 路由器 <---- 交换机 <-----多个主机
        路由器的接口有独立的网卡 每个网卡都有独立的mac地址以及ip地址 通过路由器本机进行交换数据 实现跨网通信

    4. a网里面的a1主机 想要和b网里面的b1主机通信 如何实现的？
        a：数据+目的IP+源IP+路由器a mac地址+源mac地址 给路由器 路由器a接口拿到后发现目的IP不是自己 而是另一个区域的 就给接口b 接口b就将数据+目的IP+源IP+b1mac地址+路由器b mac地址 给b1主机 b1主机接收后 发现 mac地址 和ip地址 都是给自己的 就会拿到数据了

    5. 浏览器访问服务器的过程
        1.解析域名
        2.发送请求三次握手
        3.传输数据
        4.结束后四次挥手


二、 GIL锁
    cpython解释器的弊端就是 当你用线程或协程时 即使有空闲的cpu核也不会使用 而是会并发
    如何解决？
        1.换jpython 
        2.线程或协程中的其中一个用python代码编写 另一个采用其他语言
    多线程/协程 还是会比单线程非堵塞效率高 是因为 他们会采用合适的时间切换线程锁
    cpython中 只有进程会真正的采用并行 其他都是并发
    计算型密集程序 --> 进程
    IO密集型程序 --> 协程 线程

三、深拷贝 浅拷贝
    深拷贝是从头拷到尾 隔离数据 有安全性 另一种形式的备份
    浅拷贝省空间 只会拷贝最顶层的数据 内层如果是引用 也会把引用直接拷贝下来 数据共享 
    特例： 1.元组 不可变类型 直接引用 除非嵌套可变类型时 深拷贝才会真的拷贝 其余还是引用
           2.另一种形式的浅拷贝---> 切片
           3. 字典中key是真正存在字典中 value是引用 因为字典是无序的
           4.函数传参列表 默认是浅拷贝 对参数的修改会影响原始对象 除非在传参时加上 copy.deepcopy(xxx)
